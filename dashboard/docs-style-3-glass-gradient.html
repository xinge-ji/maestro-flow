Overview

Maestro orchestrates your development workflow through structured phases, each containing tasks executed in parallel waves based on dependency analysis.

New to Maestro?
Start with the Quick Start to set up your first project in under 5 minutes.

The Pipeline

Every project follows a natural progression through these stages:

brainstorm → init → roadmap → plan → execute → verify

Getting Started

1
Install Maestro
Install globally via npm to get the maestro CLI.
2
Initialize Your Project
Run /maestro-init to create the .workflow/ structure.
3
Create a Roadmap
Define phases with /maestro-roadmap or generate specs first.
4
Plan & Execute
Use /maestro-plan then /maestro-execute for each phase.

Installation

Terminal 
# Install globally
$ npm install -g maestro-flow

# Verify installation
$ maestro --version

# Initialize a new project
$ maestro init
Prerequisites
Node.js 18+ required. Works best with Claude Code as the AI execution backend.

Key Features

FeatureDescription
Phase ManagementSequential phases with automatic state transitions
Wave ExecutionDAG-based parallel task execution within phases
MCP IntegrationExpose tools via Model Context Protocol (stdio transport)
Quality PipelineBuilt-in code review, UAT testing, debugging
Extension SystemPlugin architecture at ~/.maestro/extensions/

Configuration

TypeScript 
// maestro.config.ts
import { defineConfig } from 'maestro-flow';

export default defineConfig({
  project: {
    name: 'my-app',
    version: '1.0.0',
  },
  phases: [
    { name: 'Setup',    tasks: 3 },
    { name: 'Core API',  tasks: 5 },
    { name: 'Frontend',  tasks: 8 },
    { name: 'Deploy',    tasks: 4 },
  ],
});
Important
Always run maestro init before your first project. This creates the required .workflow/ directory.

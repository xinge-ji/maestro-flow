Overview

Maestro orchestrates your development workflow through a structured pipeline of phases, each containing tasks that are automatically planned, executed, and verified.

Tip
New to Maestro? Start with the Quick Start guide to set up your first project in minutes.

The Pipeline

Every project follows a natural flow from ideation to completion:

Pipeline 
brainstorm → init → roadmap → plan → execute → verify
     |              |                           |
     |   spec-generate (alternative)            |
     |                                  quality pipeline
     |                              review → test → debug

Installation

Install Maestro globally via npm:

bash 
# Install globally
npm install -g maestro-flow

# Verify installation
maestro --version

# Initialize a new project
maestro init
Prerequisites
Maestro requires Node.js 18+ and works best with Claude Code as the AI backend.

Quick Example

Here's a minimal example to get a project running:

typescript 
// maestro.config.ts
import { defineConfig } from 'maestro-flow';

export default defineConfig({
  project: {
    name: 'my-app',
    version: '1.0.0',
  },
  phases: [
    { name: 'Setup',    tasks: 3 },
    { name: 'Core API',  tasks: 5 },
    { name: 'Frontend',  tasks: 8 },
    { name: 'Deploy',    tasks: 4 },
  ],
});
Important
Always run maestro init before starting a new project. This creates the required .workflow/ directory structure.

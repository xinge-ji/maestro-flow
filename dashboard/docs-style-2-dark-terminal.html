What is Maestro

Maestro is a workflow orchestration CLI that structures software development into phases and tasks, with built-in AI-powered planning, execution, and verification.

It works as a layer on top of Claude Code, adding project memory, structured decomposition, and quality gates to every development cycle.

[TIP]
First time?
Jump straight to the quick-start guide and come back here for deeper understanding.

The Pipeline

Every project follows a natural execution pipeline:

maestro pipeline 
$ maestro --help

  maestro pipeline:

  brainstorm → init → roadmap → plan → execute → verify
                  |                          |
       spec-generate (alt)           quality pipeline
                                   review → test → debug

  Available commands:
    /maestro          Intelligent coordinator
    /maestro-init     Initialize project
    /maestro-plan     Plan phase execution
    /maestro-execute  Execute phase plan
    /maestro-verify   Goal-backward verification

Key Features

FeatureDescription
Phase ManagementSequential phases with auto-transitions and state tracking
Wave ExecutionDAG-based parallel task execution within each phase
MCP ServerExpose all tools via Model Context Protocol (stdio)
Quality PipelineCode review, UAT testing, and debugging automation
ExtensionsPlugin system loaded from ~/.maestro/extensions/

Installation

bash 
# install globally
$ npm install -g maestro-flow

# verify
$ maestro --version
maestro-flow v0.1.0

# initialize a project
$ maestro init
✓ Created .workflow/project.md
✓ Created .workflow/state.json
✓ Created .workflow/config.json
✓ Project initialized successfully
[!]
Prerequisite
Requires Node.js 18+. Maestro works best paired with Claude Code as the AI execution backend.

Configuration

typescript 
// maestro.config.ts
import { defineConfig } from 'maestro-flow';

export default defineConfig({
  project: {
    name: 'my-app',
    version: '1.0.0',
  },
  phases: [
    { name: 'Setup',    tasks: 3 },
    { name: 'Core API',  tasks: 5 },
    { name: 'Frontend',  tasks: 8 },
    { name: 'Deploy',    tasks: 4 },
  ],
});

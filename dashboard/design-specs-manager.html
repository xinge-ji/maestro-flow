Rule
All API routes must have request validation via Zod schemas; no unvalidated req.body access
ID
L-020
Added
15m ago
Primary
learnings.md
Secondary
quality-rules.md
Description
Every Express route handler must validate incoming request body using a Zod schema before processing. Direct access to req.body properties without validation is prohibited. Schema errors should return 400 with structured error response.
Example
const CreateUserSchema = z.object({ email: z.string().email(), name: z.string().min(2).max(100), role: z.enum(['admin', 'user']), }); router.post('/users', (req, res) => { const parsed = CreateUserSchema.safeParse(req.body); if (!parsed.success) { return res.status(400).json({ error: 'VALIDATION_ERROR', details: parsed.error.flatten(), }); } // Use parsed.data (typed!) });
